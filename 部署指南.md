# 网站部署上线完整流程
> 给另一个 AI 看，照着步骤做

---

## 📦 你的任务

把用户给你的 HTML 文件部署到 GitHub Pages，让所有人通过网址访问。如果网站需要后端数据库，配置 Supabase。

---

## 一、部署到 GitHub Pages

### 第 1 步：创建 GitHub 仓库

1. 打开 `github.com/new`
2. Repository name 填项目名（如 `my-site`）
3. 选 **Public**
4. 不要勾选任何初始化选项
5. 点绿色 **Create repository**

### 第 2 步：上传 HTML 文件

1. 点 **uploading an existing file**
2. 把 HTML 文件拖进去
3. 确保文件名是 `index.html`
4. 点绿色 **Commit changes**

### 第 3 步：开启 GitHub Pages

1. 进仓库页面 → 点 **Settings**
2. 左边菜单点 **Pages**
3. Branch 选 **master** → 点 **Save**
4. 等 30 秒 → 页面顶部出现网址：`https://用户名.github.io/仓库名/`

---

## 二、绑定自定义域名（可选）

1. 在 Pages 设置页面 → Custom domain 填你的域名
2. 点 Save → 等 DNS 验证
3. 去域名服务商（阿里云/腾讯云）添加 CNAME 记录：
```
类型：CNAME
主机记录：@（或 www）
记录值：用户名.github.io
```
4. 等几分钟生效

---

## 三、配置 Supabase 后端（如果网站需要数据库）

### 第 1 步：创建 Supabase 项目

1. `supabase.com` → **Start your project**
2. 选 Organization → New project
3. 填名字 → 设数据库密码 → 选最近区域 → Create

### 第 2 步：获取密钥

1. 进项目 → 左边 **Settings** → **API**
2. 复制 **anon public key**（以 `sb_publishable_` 开头）

### 第 3 步：建表

在 SQL Editor 里执行：

```sql
-- 用户表
CREATE TABLE user_points (
  username TEXT PRIMARY KEY,
  password TEXT,
  points INTEGER DEFAULT 600,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 邀请码表
CREATE TABLE invite_codes (
  code TEXT PRIMARY KEY,
  points INTEGER DEFAULT 600,
  used BOOLEAN DEFAULT FALSE,
  used_by TEXT,
  used_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 文案保存表
CREATE TABLE scripts (
  id TEXT PRIMARY KEY,
  username TEXT,
  script_data JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 手机号注册表
CREATE TABLE admin_users (
  username TEXT PRIMARY KEY,
  password TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 配置表
CREATE TABLE app_config (
  key TEXT PRIMARY KEY,
  value TEXT
);

-- RLS 策略（允许匿名操作）
ALTER TABLE user_points ENABLE ROW LEVEL SECURITY;
CREATE POLICY "all" ON user_points FOR ALL USING (true);

ALTER TABLE invite_codes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "all" ON invite_codes FOR ALL USING (true);

ALTER TABLE scripts ENABLE ROW LEVEL SECURITY;
CREATE POLICY "all" ON scripts FOR ALL USING (true);

ALTER TABLE admin_users ENABLE ROW LEVEL SECURITY;
CREATE POLICY "all" ON admin_users FOR ALL USING (true);

ALTER TABLE app_config ENABLE ROW LEVEL SECURITY;
CREATE POLICY "all" ON app_config FOR ALL USING (true);
```

### 第 4 步：在代码里接 Supabase

用原生 fetch（不要用 Supabase JS CDN，国内加载慢）：

```javascript
const DB = 'https://你的项目ID.supabase.co/rest/v1';
const AUTH = '你的anon key';

// 查询
async function Q(table, query='') {
  const r = await fetch(DB+'/'+table+'?'+query, {
    headers: { apikey: AUTH, Authorization: 'Bearer '+AUTH }
  });
  return r.json();
}

// 插入
async function I(table, data) {
  const r = await fetch(DB+'/'+table, {
    method: 'POST',
    headers: { apikey: AUTH, Authorization: 'Bearer '+AUTH, 'Content-Type': 'application/json', Prefer: 'return=representation' },
    body: JSON.stringify(data)
  });
  return r.json();
}

// 更新
async function U(table, query, data) {
  await fetch(DB+'/'+table+'?'+query, {
    method: 'PATCH',
    headers: { apikey: AUTH, Authorization: 'Bearer '+AUTH, 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  });
}

// 删除
async function D(table, query) {
  await fetch(DB+'/'+table+'?'+query, {
    method: 'DELETE',
    headers: { apikey: AUTH, Authorization: 'Bearer '+AUTH }
  });
}
```

## 四、AI API 接入

### DeepSeek

```javascript
async function callAI(prompt) {
  const key = 'sk-你的key'; // 建议存 localStorage 或 Supabase app_config
  const r = await fetch('https://api.deepseek.com/chat/completions', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer '+key },
    body: JSON.stringify({
      model: 'deepseek-chat',
      messages: [{ role: 'system', content: prompt }, { role: 'user', content: '创作这条口播脚本' }],
      temperature: 0.85,
      max_tokens: 3000
    })
  });
  return (await r.json()).choices[0].message.content;
}
```

## 六、技术栈总览

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 单 HTML 文件 | 无需构建，无依赖，双击打开 |
| 数据库 | Supabase | PostgreSQL REST API |
| AI | DeepSeek | 性价比最高 |
| 部署 | GitHub Pages | 免费，自动 HTTPS |
| 域名 | 阿里云/腾讯云 | 约 ¥50/年 |
